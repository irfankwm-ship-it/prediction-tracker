Methodology

Version 1.0 | January 11, 2026

Abstract

This document describes the end-to-end intelligence workflow used to produce our Global Intelligence Report. We employ a multi-stage pipeline that combines automated data collection, statistical processing, intelligence analysis, and AI-assisted report generation. Our approach prioritizes source reliability, cross-validation, and anti-hallucination safeguards to produce actionable intelligence from open sources.

1. System Architecture Overview

Our intelligence pipeline consists of six sequential stages, each building upon the previous. The system processes approximately 1,500 articles daily from 100+ sources, enriches this data with structured datasets, and synthesizes findings into readable intelligence products.

2. Data Collection (Stage 1)

The foundation of our intelligence product is comprehensive open-source data collection. We monitor multiple categories of sources:

2.1 News Sources

We maintain a curated list of 100+ news sources spanning wire services, national broadcasters, regional specialists, and subject-matter experts. Sources are selected based on:

  • Editorial independence and ownership transparency
  • Historical accuracy and correction practices
  • Geographic and topical coverage
  • Original reporting vs. aggregation ratio

2.3 Collection Schedule

News collection runs continuously with 15-minute polling intervals. Structured APIs are queried according to their update frequencies. All collected data is timestamped and stored for historical analysis.

3. Intelligence Processing (Stage 3)

3.1 Source Reliability Rating

Each source is assigned a reliability grade from A (most reliable) to F (unreliable) based on:

  • Track record: Historical accuracy, corrections issued
  • Independence: Editorial separation from state/corporate control
  • Methodology: Sourcing practices, verification standards
  • Transparency: Ownership disclosure, funding sources

3.2 Claim Extraction

We use pattern-based extraction to identify verifiable claims from article text:

  • Quote patterns: Direct statements attributed to named individuals
  • Statistical patterns: Numerical claims with units and context
  • Event patterns: Actor-verb-target structures (e.g., "Country X attacked Y")

3.3 Confidence Scoring

Each extracted fact receives a confidence score based on:

Confidence = (Source_Reliability × 0.4) + (Corroboration × 0.35) + (Recency × 0.15) + (Specificity × 0.10) Where: - Source_Reliability: A-F grade converted to 0.25-1.0 - Corroboration: Number of independent sources confirming (max 1.0 at 3+ sources) - Recency: Decay function, 1.0 for today, decreasing over time - Specificity: Presence of named actors, locations, dates

4. Statistical Processing (Stage 4)

4.1 Named Entity Recognition

We employ spaCy's English language model for entity extraction, identifying:

  • PERSON: Political leaders, military commanders, officials
  • ORG: Governments, military units, corporations, NGOs
  • GPE: Countries, states, cities
  • NORP: Nationalities, political groups, religious groups

4.2 Anomaly Detection

We maintain rolling baselines for actor mention frequency and topic prevalence. Anomalies are flagged when:

  • Actor mentions exceed 2 standard deviations from 30-day average
  • New entity pairs appear in adversarial context
  • Topic clusters show sudden growth (>50% day-over-day)

4.3 Theme Clustering

Articles are clustered using TF-IDF vectorization and k-means clustering (k=15). Cluster labels are derived from top-weighted terms and matched against predefined category keywords:

  • Conflict: attack, strike, military, troops, casualties
  • Diplomatic: summit, talks, agreement, embassy, minister
  • Economic: trade, tariff, sanctions, market, inflation
  • Political: election, vote, parliament, protest, resign

5. AI-Assisted Report Generation (Stage 6)

5.1 Model Selection

We use constrained language models for report generation. Smaller models are preferred because they:

  • Follow structured prompts more reliably
  • Generate less confabulated content
  • Are more cost-effective for high-volume generation
  • Can run locally without external API dependencies
Why not larger models? While larger language models produce more fluent prose, they are more prone to hallucination and less reliable at following strict output formats. For intelligence products where accuracy is paramount, we prioritize constraint adherence over fluency.

5.2 Anti-Hallucination Safeguards

Every AI-generated statement must cite a source fact. Our pipeline implements multiple safeguards:

  1. Citation requirement: All claims must include source references
  2. Reference validation: Cited facts are verified to exist in the source data
  3. Orphan removal: Statements without valid citations are automatically removed
  4. Consistency check: Generated content is compared against source facts for contradictions

5.3 Output Formats

Reports are generated in multiple formats:

  • HTML: Interactive web version with collapsible sections
  • PDF: Printable format for offline reading
  • JSON: Structured data for programmatic access

6. Data Enrichment Sources

6.1 Satellite Intelligence

We monitor 47 strategic locations using publicly available satellite imagery from Sentinel-2 (10m resolution) and Landsat-8. Change detection algorithms identify:

  • Construction activity at military bases
  • Ship movements at strategic ports
  • Infrastructure changes at nuclear facilities
  • Troop/vehicle concentrations

6.2 Flight Intelligence

Using ADS-B data from OpenSky Network, we track:

  • Government VIP aircraft movements
  • Military transport and patrol patterns
  • Unusual flight patterns (diversions, formation flights)
  • Diplomatic shuttle activity

6.3 Trade Intelligence

UN Comtrade data provides bilateral trade flows. We calculate:

  • Trade balance shifts between major economies
  • Commodity dependency ratios
  • Sanctions evasion indicators (trade rerouting)
  • Supply chain disruption impacts

7. Limitations and Caveats

Users should be aware of the following limitations:

  • Source bias: English-language sources are overrepresented
  • Temporal lag: Structured data may be days to weeks behind events
  • Satellite coverage: Cloud cover and imaging schedules create gaps
  • ADS-B limitations: Military aircraft often disable transponders
  • AI generation: Despite safeguards, subtle inaccuracies may occur
